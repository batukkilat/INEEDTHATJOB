Welcome to INEEDTHATJOB

Your personal automated job hunting assistant.

This app searches job boards, scores each listing against your resume, and prepares tailored applications — all automatically. You only need to review and approve before anything is sent. Nothing ever gets submitted without your say-so.

1 Installation

Never used Python or a terminal before? Start here.

Step 1 — Install Python

Windows

  1. Go to python.org/downloads and click the big yellow "Download Python 3.x.x" button.
  2. Run the installer. Important: on the first screen, check the box that says "Add Python to PATH" before clicking Install.
  3. Click "Install Now" and wait for it to finish.

macOS

  1. Open Terminal (press Cmd+Space, type "Terminal", press Enter).
  2. Type python3 --version and press Enter. If you see a version number, Python is already installed — skip to Step 2.
  3. If not, go to python.org/downloads and download the macOS installer.

Step 2 — Open a terminal and navigate to the app folder

Windows

  1. Press Win + R, type cmd, press Enter. A black window opens — that's the terminal.
  2. Find where you put the INEEDTHATJOB folder. For example, if it's on your Desktop in a folder called "INEEDTHATJOB", type: 
    cd %USERPROFILE%\Desktop\INEEDTHATJOB
  3. Press Enter. The prompt should now show the folder name.

macOS

  1. Open Terminal (Cmd+Space → "Terminal").
  2. Type cd (with a space after), then drag the INEEDTHATJOB folder from Finder into the Terminal window. The path will appear automatically. Press Enter.

💡 The terminal is just a text-based way to run programs. Every command you type here runs inside the app folder.

Step 3 — You're ready

Continue with the regular installation steps below. Use the same terminal window you just opened. Type each command and press Enter, then wait for it to finish before typing the next one.

Requirements

  • Python 3.11 or newer
  • A free Groq API key — used for resume and cover letter generation
  • An email account with SMTP enabled (Gmail recommended) — for sending applications

Getting your free Groq API key

  1. Go to console.groq.com and sign up (Google or GitHub login works).
  2. Once logged in, click API Keys in the left sidebar.
  3. Click Create API Key, name it "INEEDTHATJOB", click Create.
  4. Copy the key — it starts with gsk_. You won't see it again after closing the popup.
  5. Paste it into your .env file as shown below.

Free tier has generous daily limits — enough for dozens of applications per day.

Setup steps

  1. In your terminal, install dependencies: 
    pip install -e .
    This may take a minute. You'll see a lot of text — that's normal.
  2. Create a file named .env in the app folder (use any text editor — Notepad works) and add your Groq key: 
    GROQ_API_KEY=gsk_your_key_here
    Replace gsk_your_key_here with the key you copied. Save the file.
  3. Start the app: 
    python main.py
    On Mac, use python3 main.py if the above doesn't work.
  4. Open your browser and go to: http://localhost:8000

    Leave the terminal window open while using the app — closing it stops the server.

2 First-time setup

Do these three steps once before running your first job search.

A

Import your resume → Profile page

Upload your existing resume (PDF or DOCX). The app reads your skills, experience, and education from it. This is what all job matching and generation is based on.

You can also add/edit skills and experience manually after importing.

B

Set your target roles → Settings page

Tell the app which job titles to search for. Be specific: "Finance Staff", "Accounting Officer", "Tax Specialist". The app only fetches listings that match these titles.

Also set target locations and minimum salary while you're there.

C

Configure email → Settings → Email (SMTP)

Needed only to actually send applications. Use the Setup Guide inside the SMTP section for step-by-step instructions. You can skip this and apply manually if you prefer.

D

Add platform cookies → Settings → Platform Cookies

The scraper browses job sites as you — it needs your login session. See below for how to get these.

How to get platform cookies

A session cookie is proof that you're logged in. The scraper uses it to fetch job listings as if you were browsing normally. You grab it once from your browser — it takes about 2 minutes per platform.

Do this while logged in

You must be logged into the platform in your browser first. The cookie is only there after login.

General steps (works for all three platforms)

  1. Open the platform site in your browser (LinkedIn, Glints, or JobStreet) and log in.
  2. Press F12 to open DevTools (or right-click → Inspect).
  3. Click the Application tab (Chrome/Edge) or Storage tab (Firefox).
  4. In the left sidebar: Cookies → click the site URL.
  5. Find the cookie name in the table below, click its row, and copy the Value column.
  6. Paste it into Settings → Platform Cookies.
Platform Cookie name to copy Site to open
LinkedIn li_at linkedin.com
Glints token glints.com
JobStreet JobseekerSessionToken id.jobstreet.com

Do cookies expire?

Yes — usually after 30–90 days, or when you log out. If scraping stops working, refresh your cookies here. The app will show "0 listings found" if a cookie has expired.

Is this safe?

Cookies are saved only to .env on your computer. They never leave your machine.

4 Page-by-page guide

Dashboard

Your home base. Start job searches here, see how many jobs were found, how many are strong matches, and how many are waiting for your review.

Jobs

A full list of every job the app has found. Filter by status or platform. Click any row to see the full description and match score breakdown.

Review Queue

The most important page. Jobs that scored well land here. Generate a tailored resume, cover letter, and email per job — then approve to send, or skip. The app never submits without your approval.

History

All applications you've sent or manually applied to. Filter by status, search by company, sort by date.

Profile

Your resume data — skills, experience, education, certifications, projects. The more complete this is, the better your generated resumes and match scores will be.

Settings

Target roles, locations, salary preferences, SMTP email config, and the auto-schedule toggle. Set these up before your first search.

6 Tips & tricks

Add ATS keywords to your skills

On the Profile page, each skill has an "ATS Keywords" field. Add synonyms and abbreviations that job postings use. Example: for "Microsoft Excel", add keywords Excel, spreadsheet, vlookup, pivot table. The more keywords, the better the skill matching.

Specific achievements generate better resumes

On the Profile page, each experience has an Achievements section. Add specific, metric-driven bullets like "Reduced monthly close time by 2 days by automating reconciliation." The AI picks the most relevant ones per job — it won't invent numbers.

Generate resume before cover letter

In the Review Queue, generate the resume first. The cover letter reads the resume content and produces a more coherent, consistent application package.

Use "Apply via Platform" for jobs with their own portal

This button opens the job posting in a new tab and marks it as applied. Use it for jobs that require applying through Workday, Greenhouse, or similar systems.

Schedule daily searches so you never miss a new listing

Go to Settings → Auto-Schedule. Toggle it on and set a cron expression. Default (0 9 * * 1-5) runs every weekday at 9 AM. The app must be running for scheduled searches to fire.

Score seems low? Check the breakdown

Click any job in the Jobs list to see which sub-score dragged it down. Low Skill Match = add more keywords to your skills. Title Relevance = 0% = the job title doesn't match your target roles exactly.

7 Customization

Change the minimum score threshold

By default, only jobs scoring ≥ 55% are queued for review. Open pipeline.py and change MIN_SCORE_TO_GENERATE = 0.55 to any value between 0 and 1.

Adjust scoring weights

Open jobs/scorer.py and edit the WEIGHTS dict. Values must sum to 1.0.

WEIGHTS = {
    "skill_match":      0.35,   # ← increase to prioritize skill fit
    "experience_match": 0.25,
    "title_relevance":  0.20,
    "location_match":   0.10,   # ← decrease if location doesn't matter
    "language_match":   0.05,
    "salary_match":     0.05,
}

Swap the AI model

Add these to your .env file:

GENERATION_MODEL=llama-3.3-70b-versatile
SCORING_MODEL=llama-3.1-8b-instant

Check console.groq.com for available models and their rate limits.

Edit generation style and tone

All AI prompts are plain text files in generation/prompts/. Edit them to change the style, tone, or language of generated content — no Python needed.
